# 03. Shopify のデータモデル

分析基盤が扱う主要オブジェクトと関係。詳細なカラム定義は [`../../docs/`](../../docs/) 参照。

## 主要オブジェクト

| オブジェクト | 意味 | 分析での価値 |
|---|---|---|
| **Order (注文)** | 購入1件。明細・金額・返金・チャネル・住所を持つ | 売上・AOV・地域・返金 |
| **LineItem (注文明細)** | 注文内の商品1行 | 商品別売上・数量 |
| **Product (商品)** | 販売商品。カテゴリ・ベンダーを持つ | 商品分析 |
| **ProductVariant (バリアント)** | 商品のサイズ/色など。価格・**原価**・在庫 | 粗利・在庫 |
| **Customer (顧客)** | 購入者。生涯購入額・**メール配信同意** | セグメント・LTV・CRM |
| **Discount (割引)** | クーポン/自動割引。利用回数を持つ | 販促効果 |
| **Collection (コレクション)** | 商品のグルーピング (カテゴリ相当) | カテゴリ分析 |
| **AbandonedCheckout (放棄チェックアウト)** | カゴ落ち。未完了の購入 | ファネル・離脱 |
| **Location (ロケーション)** | 在庫/出荷拠点 | 在庫・物流 |

## 関係 (ざっくり)

```
Customer 1──* Order 1──* LineItem *──1 ProductVariant *──1 Product *──* Collection
   │                                                         
   *                                                         
Order.discountCodes ─ (コード) ─ Discount
Customer 1──* AbandonedCheckout
```

## 分析基盤での対応 (どのソースから取るか)

| Shopify | dlt 標準対応 | 本プロジェクトの取得 |
|---|---|---|
| 注文・商品・顧客 | あり | **Bulk Operations** でリッチに取得 (原価・返金・住所・同意を含む) |
| 割引 | **なし** | `discountNodes` をカスタム取得 |
| コレクション | 限定的 | Bulk でメンバーシップごと取得 |
| 放棄チェックアウト | **なし** | `abandonedCheckouts` をカスタム取得 |
| ロケーション | 限定的 | ページングで取得 |

> 「分析に有用なソースは、標準非対応でも割引と同じ要領でカスタム取得する」という方針。
